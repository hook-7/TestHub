# 前端串口配置改造说明

## 改造概述

已成功将串口配置页面从后端API调用改造为使用Web Serial API的前端直接控制。现在串口通信完全在前端进行，无需后端支持。

- `frontend/src/services/webSerial.ts` - Web Serial API服务封装

- `frontend/src/views/SerialConfig.vue` - 添加兼容性检查和UI适配
- `frontend/src/router/index.ts` - 添加测试页面路由
- `frontend/src/App.vue` - 添加测试页面导航

## 使用方法

### 1. 环境要求

- **浏览器**: Chrome 89+, Edge 89+, Opera 76+
- **协议**: 必须使用HTTPS（Web Serial API要求）
- **权限**: 需要用户手动授权串口访问

### 2. 访问页面

1. **串口配置页面**: `/serial-config`
   - 配置串口参数
   - 连接/断开串口
   - 查看已连接串口状态

2. **串口测试页面**: `/serial-test`
   - 测试Web Serial API支持
   - 测试串口连接功能
   - 测试指令发送功能

### 3. 操作步骤

#### 连接串口
1. 打开串口配置页面
2. 检查浏览器兼容性（页面顶部会显示状态）
3. 点击"刷新端口"按钮
4. 在浏览器弹出的对话框中选择串口设备
5. 配置串口参数（波特率、数据位等）
6. 点击"连接串口"按钮

#### 发送指令
1. 确保串口已连接
2. 在指令输入框中输入AT指令（如：AT+GMR）
3. 点击"发送"按钮
4. 查看通信日志中的响应

#### 发送原始数据
1. 在十六进制输入框中输入数据（如：41540D0A）
2. 点击"发送"按钮
3. 查看通信日志中的响应

## 技术实现

### Web Serial API封装

```typescript
// 主要接口
class WebSerialService {
  // 检查API支持
  isSupported(): boolean
  
  // 获取可用串口（需要用户选择）
  async getAvailablePorts(): Promise<SerialPortInfo[]>
  
  // 连接串口
  async connectSerial(config: SerialConfig): Promise<SerialConnectResponse>
  
  // 断开串口
  async disconnectSerial(serialId?: number): Promise<boolean>
  
  // 发送AT指令
  async sendATCommand(command: string, serialId?: number): Promise<RawDataResponse>
  
  // 发送原始数据
  async sendRawData(hexData: string, serialId?: number): Promise<RawDataResponse>
  
  // 获取连接状态
  async getConnectionStatus(): Promise<SerialConnectionStatus>
}
```

### 状态管理

使用Pinia store管理串口连接状态：

```typescript
// connection store
const connectionStore = useConnectionStore()

// 主要状态
connectionStore.isConnected        // 是否有连接的串口
connectionStore.connectedSerials  // 已连接的串口列表
connectionStore.selectedSerialId  // 当前选择的串口ID
connectionStore.availablePorts    // 可用串口列表

// 主要操作
connectionStore.connect(config)    // 连接串口
connectionStore.disconnect(id)     // 断开串口
connectionStore.checkStatus()      // 检查状态
```

## 注意事项

### 1. HTTPS要求
Web Serial API只能在HTTPS环境下使用，开发环境需要配置SSL证书。

### 2. 用户授权
每次连接串口都需要用户手动选择设备，无法自动连接。

### 3. 浏览器限制
- 仅支持现代浏览器
- 需要用户手动授权
- 无法枚举所有可用串口

### 4. 错误处理
- 连接失败时显示详细错误信息
- 支持重连机制
- 自动清理断开的连接
