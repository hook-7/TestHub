# 前端串口通信改造方案

## 项目概述

当前项目是一个工业HMI系统，采用前后端分离架构：
- **后端**: FastAPI + Python，负责串口通信、WebSocket实时通信、数据管理
- **前端**: Vue 3 + TypeScript + Element Plus，负责用户界面和交互

## 改造目标

将串口通信从后端迁移到前端，实现：
1. 前端直接控制串口设备
2. 保持现有页面和功能不变
3. 减少后端复杂度
4. 提高系统响应速度

## 技术方案

### 1. 前端串口通信技术选型

#### 方案A: Web Serial API (推荐)
- **优势**: 浏览器原生支持，无需额外依赖
- **兼容性**: Chrome 89+, Edge 89+, Opera 76+
- **适用场景**: 现代浏览器环境，工业应用

#### 方案B: Electron + Node.js SerialPort
- **优势**: 跨平台兼容性好，功能完整
- **劣势**: 需要打包为桌面应用
- **适用场景**: 需要支持旧版浏览器

#### 方案C: WebUSB + 自定义驱动
- **优势**: 更底层的控制
- **劣势**: 开发复杂度高，需要设备支持
- **适用场景**: 特殊硬件需求

**推荐使用方案A (Web Serial API)**，因为：
1. 无需额外打包
2. 浏览器原生支持
3. 开发维护简单
4. 符合现代Web标准

### 4. 技术实现细节

#### Web Serial API 封装

```typescript
export class WebSerialService {
  private ports: Map<number, SerialPort> = new Map()
  private portConfigs: Map<number, SerialConfig> = new Map()
  private dataCallbacks: Map<number, (data: string) => void> = new Map()
  
  async getAvailablePorts(): Promise<SerialPortInfo[]> {
    // 请求用户选择串口
    const port = await navigator.serial.requestPort()
    return [{
      device: port.getInfo().usbVendorId?.toString() || 'unknown',
      name: port.getInfo().usbProductId?.toString() || 'unknown',
      description: 'USB Serial Device',
      hwid: 'unknown',
      manufacturer: 'unknown'
    }]
  }
  
  async connect(config: SerialConfig): Promise<number> {
    const port = await navigator.serial.requestPort()
    await port.open({
      baudRate: config.baudrate,
      dataBits: config.bytesize,
      parity: config.parity,
      stopBits: config.stopbits,
      flowControl: 'none'
    })
    
    const serialId = this.getNextSerialId()
    this.ports.set(serialId, port)
    this.portConfigs.set(serialId, config)
    
    // 启动数据读取
    this.startReading(serialId, port)
    
    return serialId
  }
  
  private async startReading(serialId: number, port: SerialPort) {
    const reader = port.readable?.getReader()
    if (!reader) return
    
    while (true) {
      const { value, done } = await reader.read()
      if (done) break
      
      const data = new TextDecoder().decode(value)
      const callback = this.dataCallbacks.get(serialId)
      if (callback) {
        callback(data)
      }
    }
  }
  
  async sendCommand(command: string, serialId?: number): Promise<RawDataResponse> {
    const port = this.ports.get(serialId || 1)
    if (!port) throw new Error('Port not connected')
    
    const writer = port.writable?.getWriter()
    if (!writer) throw new Error('Port not writable')
    
    const data = new TextEncoder().encode(command + '\r\n')
    await writer.write(data)
    writer.releaseLock()
    
    return {
      serial_id: serialId || 1,
      sent_data: command,
      received_data: 'Command sent',
      timestamp: Date.now()
    }
  }
}
```

### 9. 风险评估

#### 技术风险
- **Web Serial API兼容性**: 仅支持现代浏览器
- **权限管理**: 需要用户手动授权串口访问
- **错误处理**: 串口通信错误处理更复杂

#### 解决方案
- 提供降级方案，不支持时回退到后端
- 完善错误提示和用户引导
- 加强异常处理和重连机制
