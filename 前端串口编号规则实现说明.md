# 前端串口编号规则实现说明

## 概述

按照后端串口编号规则，在前端实现了智能的串口ID分配策略，确保ID分配的一致性和可复用性。

## 后端编号规则

根据后端 `SerialDriver` 类的 `get_next_serial_id()` 方法：

```python
def get_next_serial_id(self) -> int:
    """获取下一个可用的串口ID（从1开始，复用断开的ID）"""
    # 如果没有连接，从1开始
    if not self.connections:
        return 1
        
    # 寻找最小的未使用ID，从1开始
    used_ids = set(self.connections.keys())
    for serial_id in range(1, max(used_ids) + 2):
        if serial_id not in used_ids:
            return serial_id
    
    # 理论上不会到达这里，但为了安全起见
    return max(used_ids) + 1
```

## 前端实现

### 1. 修改前（简单递增）

```typescript
private getNextSerialId(): number {
  return this.nextSerialId++
}
```

**问题**：
- ID只增不减，无法复用
- 断开连接后ID不会重新分配
- 与后端逻辑不一致

### 2. 修改后（智能分配）

```typescript
private getNextSerialId(): number {
  // 如果没有连接，从1开始
  if (this.ports.size === 0) {
    return 1
  }
  
  // 寻找最小的未使用ID，从1开始
  const usedIds = Array.from(this.ports.keys())
  for (let serialId = 1; serialId <= Math.max(...usedIds) + 1; serialId++) {
    if (!usedIds.includes(serialId)) {
      return serialId
    }
  }
  
  // 理论上不会到达这里，但为了安全起见
  return Math.max(...usedIds) + 1
}
```

## 编号规则详解

### 1. 基本原则
- **从1开始**：第一个串口连接分配ID 1
- **连续分配**：按顺序分配可用ID
- **复用机制**：断开连接后，ID可以被重新使用
- **最小优先**：优先分配最小的可用ID

### 2. 分配场景

#### 场景1：首次连接
```
连接串口A → 分配ID: 1
连接串口B → 分配ID: 2
连接串口C → 分配ID: 3
```

#### 场景2：断开中间连接
```
当前状态: [1, 2, 3]
断开ID 2 → 状态: [1, 3]
连接串口D → 分配ID: 2 (复用)
```

#### 场景3：断开第一个连接
```
当前状态: [1, 2, 3]
断开ID 1 → 状态: [2, 3]
连接串口E → 分配ID: 1 (复用)
```

#### 场景4：全部断开后重新连接
```
当前状态: [1, 2, 3]
全部断开 → 状态: []
连接串口F → 分配ID: 1 (重新开始)
```

### 2. ID分配算法
```typescript
private getNextSerialId(): number {
  // 1. 检查是否有连接
  if (this.ports.size === 0) {
    return 1  // 无连接时从1开始
  }
  
  // 2. 获取已使用的ID列表
  const usedIds = Array.from(this.ports.keys())
  
  // 3. 从1开始寻找最小可用ID
  for (let serialId = 1; serialId <= Math.max(...usedIds) + 1; serialId++) {
    if (!usedIds.includes(serialId)) {
      return serialId  // 找到第一个可用ID
    }
  }
  
  // 4. 安全回退（理论上不会执行）
  return Math.max(...usedIds) + 1
}
```

### 3. 连接管理
```typescript
async connectSerial(config: SerialConfig): Promise<SerialConnectResponse> {
  // 1. 请求用户选择串口
  const port = await navigator.serial.requestPort()
  
  // 2. 分配串口ID
  const serialId = this.getNextSerialId()
  
  // 3. 保存连接信息
  this.ports.set(serialId, port)
  this.portConfigs.set(serialId, config)
  
  // 4. 返回连接结果
  return {
    serial_id: serialId,
    port: 'user-selected',
    message: `串口连接成功，分配ID: ${serialId}`
  }
}
```
