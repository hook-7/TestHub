# 串口参数修复说明

## 问题描述

在连接串口时出现错误：
```
串口连接失败: Failed to execute 'open' on 'SerialPort': Failed to read the 'parity' property from 'SerialOptions': The provided value 'N' is not a valid enum value of type ParityType.
```

## 问题原因

Web Serial API的`parity`属性需要使用特定的枚举值：
- `'none'` - 无校验
- `'even'` - 偶校验  
- `'odd'` - 奇校验

而我们的代码中使用了：
- `'N'` - 无校验
- `'E'` - 偶校验
- `'O'` - 奇校验

## 解决方案

### 1. 添加参数转换函数

在`WebSerialService`类中添加了`convertParity`方法：

```typescript
private convertParity(parity: string): 'none' | 'even' | 'odd' {
  switch (parity.toUpperCase()) {
    case 'N':
      return 'none'
    case 'E':
      return 'even'
    case 'O':
      return 'odd'
    default:
      console.warn(`Unknown parity value: ${parity}, using 'none'`)
      return 'none'
  }
}
```

### 2. 修改连接逻辑

在`connectSerial`方法中使用转换函数：

```typescript
const portConfig: SerialPortOpenOptions = {
  baudRate: config.baudrate,
  dataBits: config.bytesize as 7 | 8,
  parity: this.convertParity(config.parity), // 使用转换函数
  stopBits: config.stopbits as 1 | 2,
  flowControl: 'none'
}
```

### 3. 保持用户界面不变

用户界面仍然使用'N'、'E'、'O'作为选项值，但在内部转换为Web Serial API需要的格式。

- 选择校验位：无校验/偶校验/奇校验
- 系统自动转换为Web Serial API格式
- 连接时使用正确的参数

### 3. 错误处理
如果遇到参数相关错误，检查：
- 浏览器控制台是否有警告信息
- 参数值是否正确
- Web Serial API是否支持该参数组合
